Write-up: Network Enum

Easy

The primary objective of this lab was to establish the basics of infrastructure reconnaissance, a critical phase before any exploitation.

1. Active Reconnaissance

We started with an exhaustive port scan using **Nmap** to discover exposed services on the target:

nmap -sC -sV -p- 10.10.X.X -oN initial_scan.txt

Ports 22 (SSH) and 80 (HTTP) were discovered. The -sV flag allowed us to identify the exact version of the running web server, which is the vital first step in searching for known vulnerabilities (CVEs), while -sC ran default enumeration scripts.

2. Result Analysis

With the obtained information, the next logical step is directory enumeration on port 80 using tools like Gobuster or Feroxbuster to map the web attack surface.

Write-up: Login Bypass

Medium

The objective of this machine was to compromise a web authentication panel vulnerable to brute force attacks, demonstrating the importance of robust password policies and rate limiting.

1. Vector Identification

After enumerating the web application, we located a login form. When attempting basic SQL injections (' OR 1=1 --), we verified the inputs were sanitized. However, there were no lockout mechanisms after multiple failed attempts.

2. Exploitation

We proceeded to perform a dictionary attack using **Hydra**, intercepting the POST request to analyze the required parameters.

hydra -l admin -P /usr/share/wordlists/rockyou.txt 10.10.X.X http-post-form "/login.php:user=^USER^&pass=^PASS^:F=Incorrect login"

By using the rockyou wordlist, we successfully obtained valid credentials and breached the perimeter, gaining access to the administrator dashboard.
